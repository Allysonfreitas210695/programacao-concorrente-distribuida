(a) Qual é o escalonamento padrão?

O escalonamento padrão confirmado pela execução é STATIC com
chunk = ceil(n / num_threads).

Evidência direta dos resultados:
  - 8 iters / 2 threads → chunk = 4  (cada thread recebe 4 iterações contíguas)
  - 8 iters / 4 threads → chunk = 2  (cada thread recebe 2 iterações contíguas)
  - 16 iters / 4 threads → chunk = 4 (blocos de 4 contíguos)
  - 32 iters / 8 threads → chunk = 4 (blocos de 4 contíguos)

Padrão: thread t recebe as iterações [t*chunk .. (t+1)*chunk - 1].
As iterações são contíguas, nunca intercaladas — isso é STATIC puro.

POR QUE O PADRÃO É STATIC?
O escalonamento STATIC divide o trabalho ANTES da execução, calculando
uma única vez quais iterações cada thread vai processar. Isso elimina
todo o overhead de coordenação em tempo de execução. Para laços com
custo uniforme por iteração (como a maioria dos científicos), é a
escolha ótima e por isso foi adotado como padrão pelo OpenMP.
